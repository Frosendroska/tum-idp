ends before any sustained-load throttle can engage.

=== CROSS-PLOT FINDINGS & KEY OBSERVATIONS ===

1. NO THROTTLING CLIFF: Unlike the r8gd.4xlarge runs, c5n.9xlarge shows no abrupt throughput
   collapse. All phases operate in the 38–41 Gbps band. The short run duration (11.5 min)
   and ramp-only design mean burst credits were never depleted.

2. MONOTONIC THROUGHPUT DECLINE WITH CONCURRENCY: Throughput decreases with each concurrency
   step: 324→41.2 Gbps, 432→37.4 Gbps, 540→36.9 Gbps. The decline is steeper between
   324 and 432 (~3.8 Gbps) than between 432 and 540 (~0.5 Gbps), suggesting saturation
   above 432 connections.

3. MONOTONIC LATENCY INCREASE WITH CONCURRENCY: Mean latency increases ~2,100–2,300 ms per
   concurrency step: 324→5,942 ms, 432→8,254 ms, 540→10,378 ms. Combined with the
   throughput decline, this indicates the c5n.9xlarge is operating near or above its
   sustainable I/O saturation point at 432+ connections.

4. HIGH ABSOLUTE THROUGHPUT: ~40 Gbps average is the highest of any instance in this
   experiment series. The c5n.9xlarge has a 50 Gbps network card; this run achieved ~80%
   of the theoretical maximum NIC bandwidth.

5. HTTP 200 vs HTTP 206: This run uses full-object GET (HTTP 200), not range reads (HTTP 206)
   as used in the r8gd.4xlarge runs. This difference in request type must be accounted for
   when comparing latency and throughput values across instance types.

6. EFFICIENCY COLLAPSE AT HIGH CONCURRENCY: Per-connection efficiency drops from 0.127 Gbps
   at 324 connections to 0.068 Gbps at 540 — a 46% reduction. The optimal concurrency for
   this instance is 324 or below.

7. PERFECT RELIABILITY: 100% success rate, no retries recorded, single HTTP status code (200).
   R2 handles this load without any errors even at 540 concurrent connections and ~40 Gbps.

8. END-OF-RUN DRAIN ARTEFACT: Both throughput plots show a near-zero or sharp drop in the
   final 1–2 windows of ramp_3. This is a run-termination artefact (last in-flight requests
   completing with no new ones queued), not a performance event.
